## v1.4 变更详情（2026-03-22）

### 1. stock_daily表新增quality_flag字段
- **变更前**：无质量标记字段
- **变更后**：新增quality_flag字段，标记数据质量等级
  - good：正常数据，直接入库
  - warn：可疑数据，标记后入库
  - error：异常数据，入异常表待处理

### 2. 主需求文档结构调整
- **变更前**：版本变更记录在文档末尾，且包含详细变更内容
- **变更后**：
  - 版本变更记录移至主需求文档第一章（仅保留版本历史表格）
  - 详细变更内容拆分至本CHANGELOG.md文档
  - 防止主需求文档过度膨胀

## v1.3 变更详情（2026-03-22）

### 1. 新增业务流程图章节
- **变更前**：无流程图
- **变更后**：新增6个详细流程图
  - 10.1 股票数据同步主流程（含异常处理）
  - 10.2 每日增量更新流程（含异常处理）
  - 10.3 数据质量校验流程（含异常处理）
  - 10.4 异常数据处理流程
  - 10.5 断点续传详细流程
  - 10.6 数据同步报告内容说明

### 2. 新增数据同步报告功能（F9）
- **变更前**：无报告功能
- **变更后**：每次运行生成详细数据报告，包括：
  - 执行概况（运行时间、持续时长、同步类型）
  - 数据统计（处理总数、成功数、失败数、新增/更新记录数）
  - 异常汇总（网络/数据/业务异常数量）
  - 失败详情（股票代码、原因、错误信息、重试次数）
  - 质量分析（完整率、涨跌幅校验通过率、OHLC校验通过率）
  - 性能指标（平均/最大/最小耗时、API调用次数）
  - 历史趋势（与上次对比、数据量变化）

### 3. 异常处理策略明确
- **变更前**：异常未明确处理方式
- **变更后**：
  - 网络异常：重试机制（指数退避），最大重试5次
  - 数据异常：可修复→自动修复+入库，不可修复→入异常表
  - 业务异常：退市股票→标记跳过，ST股票→标记+允许扩大涨跌幅

### 4. 断点续传机制完善
- **变更前**：简单断点续传
- **变更后**：
  - 记录失败详情到sync_error表
  - 错误分类处理（超时、API限流、数据不存在、服务器错误）
  - 重试机制与断点恢复结合

### 5. 新增数据模型
- **变更前**：仅有3个表（stock_info、stock_daily、sync_status）
- **变更后**：新增2个表
  - sync_error：同步异常记录表
  - sync_report：同步报告表

## v1.1 变更详情（2026-03-22）

### 1. stock_daily表索引补充
- **变更前**：仅有联合唯一索引 `(trade_date, stock_code)`
- **变更后**：
  - 联合唯一索引：`(trade_date, stock_code)`
  - 新增普通索引：`stock_code`

### 2. 数据源策略明确
- **变更前**：未明确数据源策略
- **变更后**：优先统一从一个数据源获取，保障数据一致性，记录data_source字段便于后续审计和切换

### 3. 增加小批量验证阶段
- **变更前**：直接全量获取
- **变更后**：
  - Phase 0：选取3-5只股票验证20年数据
  - Phase 1：验证通过后再全量获取

### 4. 存储方案明确
- **变更前**：未确定存储方案
- **变更后**：使用ClickHouse列式存储

### 5. 技术环境更新
- **变更前**：未明确
- **变更后**：
  - Linux虚拟机
  - 8核CPU
  - 16GB内存
  - 200GB硬盘
