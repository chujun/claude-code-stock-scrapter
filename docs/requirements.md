## 一、项目概述

### 1.1 项目背景

构建一个A股股票历史数据爬虫系统，用于本地量化策略验证。系统需获取A股（沪深交易所）所有股票近20年的完整价格数据，为量化策略回测提供数据支持。

### 1.2 项目目标

1. 获取A股全量股票（包含已退市股票）的历史日线数据
2. 提供高质量的前复权数据，确保股价走势连续性
3. 支持每日增量更新，保持数据时效性
4. 存储于ClickHouse，支持后续扩展分析需求
5. **支持本地量化策略验证**：为本地量化策略提供数据查询接口，便于运行策略回测和验证

## 五、功能需求

### 5.1 核心功能

| 编号 | 功能 | 优先级 | 说明 |
|------|------|--------|------|
| F1 | 获取A股股票列表 | P0 | 获取所有股票基本信息 |
| F2 | 爬取单只股票历史日线数据 | P0 | 前复权数据 |
| F3 | **小批量验证** | P0 | 先验证3-5只股票20年数据，确认成功后再全量 |
| F4 | 全量数据同步 | P0 | 首次全量爬取历史数据 |
| F5 | 增量数据同步 | P0 | 每日定时更新最新数据 |
| F6 | 爬取大盘指数数据 | P1 | 上证/深证/创业板指数 |
| F7 | 爬取分红送股数据 | P2 | 复权因子计算辅助 |
| F8 | 断点续传 | P0 | 支持失败后从断点恢复 |

### 5.2 数据源策略

| 策略 | 说明 |
|------|------|
| **单一数据源优先** | 优先统一从一个数据源获取，保障数据一致性 |
| 数据源标识 | 记录data_source字段，便于后续审计和切换 |
| 备选数据源 | 仅在主数据源不可用时切换 |

### 5.3 性能需求

| 指标 | 目标值 |
|------|--------|
| 小批量验证 | < 1小时（3-5只股票） |
| 全量爬取时间 | < 8小时（分批执行） |
| 单只股票请求间隔 | 1-2秒（避免限速） |
| 增量更新 | < 30分钟（每日） |
| 数据延迟 | 当日收盘后1小时内 |

### 5.4 质量需求

| 指标 | 要求 |
|------|------|
| 数据完整率 | > 99.5% |
| 数据准确率 | > 99.9% |
| 涨跌幅校验 | ±10%以内 |
| 成功率 | > 95% |

## 八、实施策略

### 8.1 验证阶段（Phase 0）

**目标**：验证数据获取可行性

1. 选取3-5只不同市场的股票
   - 如：600000（上海）、000001（深圳）、300750（创业板）、688001（科创板）
2. 获取2004年至今的所有历史数据
3. 验证数据完整性、准确性
4. 验证ClickHouse存储和查询

**成功标准**：
- 数据完整率 > 99%
- 涨跌幅校验通过
- 查询响应正常

### 8.2 全量阶段（Phase 1）

验证通过后，启动全量数据爬取。
